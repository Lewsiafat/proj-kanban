# Threads 發文操作手冊 — proj-kanban（build in public）

> 搭配計畫文件 `2026-06-30-threads-post-plan-design.md` 使用。
> 時間軸以「發文時刻」為原點 **T=0**;`T-1d` = 發文前一天,`T+30m` = 發文後 30 分鐘。
> 每一步拆成 **When / What / Why**。

---

## 階段 0｜發文前 1 天(T-1d):備料

### 步驟 0-1 — 設定個人檔案連結
- **When**:T-1d(最晚 T-2h)
- **What**:到 Threads 個人檔案,把 bio 連結設成 GitHub repo(`github.com/Lewsiafat/proj-kanban`)。bio 一句話寫清楚你是誰 + 在做什麼(例:「用 Claude Code build in public｜做了一個給 AI 用的看板」)。
- **Why**:主貼**不能放外部連結**(會被降觸及),所以所有想點連結的人都會被導去個人檔案。檔案連結沒設好 = 流量白白流失。bio 一句話決定「點進來的人要不要追蹤」。

### 步驟 0-2 — 素材定稿
- **When**:T-1d
- **What**:確認兩個檔案就緒——主貼配圖 `shot-project-dark.png`、留言 2 的 `kanban-agent-update.gif`。在手機上各看一遍(Threads 是手機為主的場域),確認文字在小螢幕讀得清楚。
- **Why**:帶視覺的貼文觸及明顯較高;但素材要在**手機尺寸**下成立,桌機看起來漂亮不代表手機可讀。GIF 尤其要確認「卡片移動」一眼看得懂。

### 步驟 0-3 — 文案搬進手機備忘錄
- **When**:T-1d
- **What**:把主貼 + 4 則留言文案,從計畫文件複製到手機備忘錄,**一則一段**排好。
- **Why**:發文當下要「主貼一出、留言馬上接龍」,現場打字會拖慢節奏、錯過黃金期。預先排好可一鍵貼上。

### 步驟 0-4 — 約 2–3 位早期互動的朋友
- **When**:T-1d 預告,T=0 通知
- **What**:私訊 2–3 位活躍的朋友,告訴他們你大概幾點會發,請他們發文後**前 15 分鐘**幫忙留個真實留言(不是只按讚,最好是一句回應或提問)。
- **Why**:Threads 演算法看「**前期互動速度**」——發文後短時間內就有留言,系統會判定「這篇有料」,進而推給更多陌生人。冷啟動的破冰互動是放大器。

---

## 階段 1｜發文當下(T=0):投放

### 步驟 1-1 — 選對時段發文
- **When**:平日 **21:30–22:30**,或週日晚間(台灣繁中圈最活躍)
- **What**:在這個時間區間內按下發布。避開上班尖峰、深夜。
- **Why**:黃金一小時的互動量決定演算法初判,所以要在「最多人線上滑」的時段啟動,確保前 60 分鐘有足夠人看到、有人留言。

### 步驟 1-2 — 發主貼(角度 A + 配圖)
- **When**:T=0
- **What**:貼上主貼文案,附上 `shot-project-dark.png`。**檢查第一行**是「我做了一個看板。」獨立成行——這行就是預覽,決定點開率。**主貼零連結**。
- **Why**:第一行=feed 預覽,curiosity gap 要在這裡製造「蛤?」的停頓。連結放主貼會直接被降觸及,所以連結一律往後放。

### 步驟 1-3 — 立刻接龍 4 則留言
- **When**:T=0 ~ T+2m(主貼一發出**馬上**做)
- **What**:用兩種方式之一把 4 則留言串成 thread:
  - **(推薦)發文前在編輯器用「串接」功能**(撰寫畫面的鏈結 /「加入串文」鈕)把主貼 + 4 則疊好,一次發布;或
  - 主貼發出後,**回覆自己的貼文**,依序貼上留言 1→2→3→4(留言 2 記得附 `kanban-agent-update.gif`)。
- **Why**:self-reply thread 把細節往後藏,**拉長停留時間**、並一次製造多個「可被回覆/按讚」的位置。留言 4 的提問是引導陌生人留言的鉤子,必須在第一時間就到位。**連結只出現在留言 4 / 個人檔案**。

---

## 階段 2｜黃金一小時(T=0 ~ T+60m):放大

### 步驟 2-1 — 守住前 60 分鐘,逐則回覆
- **When**:T=0 ~ T+60m,**人不要離開**
- **What**:每一則進來的留言都回——而且**回得有內容**(反問、補充、感謝+追問),不要只回「謝謝」。
- **Why**:每一次「你回覆別人」都會再觸發一波觸及,而且「留言數+對話長度」是 Threads 最看重的訊號。黃金一小時的對話密度,直接決定這篇會不會被推到更大的池子。

### 步驟 2-2 — 用提問驅動更多留言
- **When**:T+0 ~ T+60m
- **What**:回覆別人時順手丟回一個小問題(例:「你都開幾個 agent 同時跑?」)。若某則留言引發共鳴,把它的觀點再延伸成一句,釘在對話裡。
- **Why**:留言比讚、比轉發都更有權重。把「單向被讚」轉成「雙向對話」,是這個階段唯一要做的事。

### 步驟 2-3 — 啟動朋友破冰
- **When**:T+0 ~ T+15m
- **What**:通知步驟 0-4 約好的朋友進來留言。
- **Why**:確保前 15 分鐘的互動速度達標,給演算法「這篇值得推」的早期訊號。

---

## 階段 3｜第一天(T+1h ~ T+24h):續航

### 步驟 3-1 — 持續回覆、不要消失
- **When**:發文後當晚 + 隔天早上、午休各回一輪
- **What**:把後續累積的留言補回完。
- **Why**:Threads 的貼文長尾比 X/IG 久,隔天仍可能因新留言再被推一波;持續回覆能延長這篇的生命週期。

### 步驟 3-2 — 觀察「哪一則留言最熱」
- **When**:T+24h
- **What**:看 4 則留言中哪一則互動最高(通常是痛點留言 1 或提問留言 4)。記下來。
- **Why**:最熱的那則,代表受眾真正在意的切角——它就是你**下一篇主貼的 hook 候選**。

---

## 階段 4｜復盤與系列化(T+1d ~ T+7d)

### 步驟 4-1 — 數據復盤
- **When**:T+2d 或 T+7d
- **What**:記錄這篇的:留言數、轉發數、個人檔案點擊、追蹤增長、連結點擊。對照下表判斷成效。
- **Why**:有基準才知道下一篇要調什麼。對 build-in-public,**留言數與追蹤增長**是主指標,讚只是參考。

| 指標 | 觀察重點 | 不如預期時的調整 |
|---|---|---|
| 留言數 | 主指標,演算法權重最高 | 提問換更具體/更挑釁的問法 |
| 個人檔案點擊 | 有沒有人想「認識你」 | 主貼金句/人設再強化 |
| 追蹤增長 | build-in-public 的核心成果 | bio 與第一印象優化 |
| 連結點擊 | 有沒有人想看 repo | 留言 4 的 CTA 講更清楚的好處 |

### 步驟 4-2 — 決定是否系列化
- **When**:T+7d
- **What**:若主貼表現好,把最熱的留言切角擴寫成**下一篇獨立主貼**(角度 B「同時開 5 個 agent」或角度 C「我跟 Claude Code 一起蓋」),沿用同一套素材節奏。
- **Why**:單篇衝量後,系列化才能把一次性流量變成**穩定的個人品牌追蹤者**,這正是這次設定的首要目標。
