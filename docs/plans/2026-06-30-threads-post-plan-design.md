# Threads 貼文計畫 — proj-kanban（build in public）

**日期**:2026-06-30
**目標平台**:Threads
**語言**:繁體中文為主(保留 Claude Code / AI agent / kanban 等英文關鍵字)
**首要目標**:個人品牌 / build in public(主指標:留言數、追蹤增長)
**產出形態**:單發主貼(高強度 hook)+ 自留言串接龍(self-reply thread)

---

## 1. 核心敘事

同一個專案、三個事實層,用一篇貼文一次講完:

- **問題**:同時開多個 AI agent 跑任務時,人類記不得誰做完、誰卡住、誰在等回話。
- **概念**:做了一個 kanban,欄位=專案、卡片=狀態更新;**agent 跑任務時自己透過 REST API 把進度寫回看板**,還打包成 Claude Code plugin 讓 agent 裝了就會用。
- **過程(build in public)**:整個專案用 Claude Code 一行一行蓋出來(Express + SQLite + 一頁 vanilla JS),每版都有 spec / walkthrough。

金句(轉發記憶點):**「給 AI 用的看板 / 人看畫面,AI 自己更新。」**

---

## 2. Hook 角度決策

| 角度 | 切角 | 採用 |
|---|---|---|
| **A 反直覺好奇**「這看板不是給人用的」 | curiosity gap,衝停留+留言,且自帶金句 | ✅ **主貼** |
| B 痛點共鳴「同時開 5 個 agent 的崩潰」 | 共鳴強但受眾較小、開場不夠金句 | 融入留言 1 |
| C build-in-public 過程「我跟 Claude Code 一起蓋的」 | 最貼個人品牌但陌生人停留誘因弱 | 融入留言 3 |

策略:A 衝觸及,B/C 在留言串補深度——一篇同時做到衝量 + 立個人品牌。

## 4. 主貼文案(角度 A,可直接複製)

```
我做了一個看板。

但使用者不是人,是 AI。

起因很蠢:我同時開了好幾個 AI agent 在跑不同任務,結果根本記不得誰做完、誰卡住、誰在等我。視窗開一堆,腦袋一片空白。

所以我做了一個 kanban——欄位是專案、卡片是進度。重點是:agent 跑任務時,會自己透過 API 把狀態寫回看板。

我只要打開瀏覽器,就看得到誰在進行、誰卡住、誰做完。

人看畫面,AI 自己更新。

(完整運作、demo,還有它怎麼用 Claude Code 一路蓋出來的 👇)
```

要點:1–2 行 curiosity gap(獨立斷行佔滿手機預覽)；第 3 行立刻接地氣講真痛點避免被當噱頭；倒數第二行金句；結尾 👇 導去留言串且不放連結。

## 5. 留言串接龍(4 則 self-reply,逐則複製)

**留言 1 — 痛點放大(角度 B)**
```
講具體一點:你有沒有同時開 4、5 個 agent,一個在重構、一個在寫測試、一個在查 bug?
跑到後來,你根本分不清哪個跑完了、哪個在等你。
我要的其實只是一個地方,能一眼看完全部任務狀態。就這樣。
```

**留言 2 — 概念 + demo(放 GIF)**
```
運作其實很單純:
欄位 = 專案,卡片 = 一則狀態更新(active / pending / done / blocked)。
agent 做完一段就呼叫 REST API 改卡片狀態,看板即時反映。
我還把「怎麼用這個 API」打包成一個 Claude Code plugin —— agent 裝了就會自己用,我不用教它。
👉 [配 GIF:卡片從 in-progress 自己滑到 done]
```

**留言 3 — build in public(角度 C)**
```
還有一件事:這整個專案是我用 Claude Code 一行一行蓋出來的。
沒有框架、沒有打包工具 —— 一個 Express + SQLite + 一頁 vanilla JS。
雙語 UI、淺/深主題、動畫… 每個版本我都留了 spec 跟 walkthrough。對我來說它本身就是個「人 + AI 一起開發」的實驗。
```

**留言 4 — CTA + 提問(鉤留言)**
```
它是開源的(MIT),也能自己 self-host。連結放在我個人檔案 🔗
想問大家:你平常怎麼追蹤 AI agent 的工作進度?還是…根本沒在追,靠記憶硬撐?😅
(我賭很多人是後者)
```

遞進:痛點 → 怎麼運作 → 怎麼做出來 → 行動+提問;連結只在此/個人檔案;末則「賭你也是」式提問最易引發「+1 我也是」。
